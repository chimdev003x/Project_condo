สร้างเว็บไซต์อสังหาริมทรัพย์สำหรับค้นหาและประกาศขาย/เช่าคอนโด ชื่อเว็บไซต์ **“Condo Finder”** โดยออกแบบให้เป็นเว็บไซต์แนว Property Listing Platform คล้ายเว็บไซต์ประกาศขายคอนโดมืออาชีพ เช่น DDProperty แต่ต้องใช้เอกลักษณ์แบรนด์ของ Condo Finder เป็นหลัก

## 1. Brand Identity

**Website Name:** Condo Finder
**Tagline:** ชี้เป้าคอนโดเด็ด

ใช้โลโก้ Condo Finder ที่แนบมาเป็นแนวทางในการออกแบบ Mood & Tone ของเว็บไซต์

### Theme Color

ใช้สีเว็บไซต์ให้สอดคล้องกับโลโก้ โดยเน้นโทนสะอาด ทันสมัย และน่าเชื่อถือ

สีหลักที่ต้องใช้:

* Primary Color: Teal / เขียวอมฟ้า เช่น `#0B6F6F`
* Secondary Color: Mint Green เช่น `#1FAF9C` หรือ `#A8E6D1`
* Accent Color: Soft Gold เช่น `#D8B95A`
* Background Color: White / Off-white / Light Gray เช่น `#FFFFFF`, `#F7FAF9`
* Text Color: Dark Teal / Charcoal เช่น `#073B3A`, `#1F2937`

## 2. ภาษาที่ใช้ในเว็บไซต์

เว็บไซต์ต้องใช้ **ภาษาไทยเป็นหลัก** เพราะกลุ่มเป้าหมายคือผู้ใช้งานในประเทศไทย

ข้อกำหนดด้านภาษา:

* ใช้ภาษาไทยที่อ่านง่าย สุภาพ และเป็นมืออาชีพ
* ข้อความต้องเข้าใจง่าย ไม่ยาวเกินไป
* ใช้คำที่คุ้นเคยกับเว็บไซต์อสังหาริมทรัพย์ เช่น “ขายคอนโด”, “เช่าคอนโด”, “ทำเล”, “ราคา”, “ห้องนอน”, “ห้องน้ำ”, “พื้นที่ใช้สอย”
* สามารถมีภาษาอังกฤษประกอบในบางจุดได้ เช่น Login, Register, New Project แต่เมนูหลักและเนื้อหาหลักควรเป็นภาษาไทย
* หลีกเลี่ยงคำแปลที่แข็งหรือไม่เป็นธรรมชาติ

ตัวอย่างข้อความหลัก:

* “ค้นหาคอนโดที่ใช่ ในทำเลที่คุณต้องการ”
* “รวมประกาศขายและให้เช่าคอนโดคุณภาพ ค้นหาง่าย เปรียบเทียบสะดวก”
* “ลงประกาศฟรี”
* “ดูรายละเอียด”
* “ติดต่อเจ้าของประกาศ”
* “โครงการใหม่”
* “ทำเลยอดนิยม”

## 3. Design Style

ออกแบบเว็บไซต์ให้มีลักษณะ:

* Modern
* Clean
* Professional
* Minimal Luxury
* Trustworthy
* Responsive
* User-friendly

เว็บไซต์ควรดูสว่าง โปร่ง สะอาด ใช้งานง่าย และให้ความรู้สึกน่าเชื่อถือ เหมาะกับธุรกิจซื้อ ขาย และเช่าคอนโด

ใช้รูปภาพประกอบเป็นภาพคอนโด ห้องพัก อาคารสูง หรือวิวเมือง โทนสว่าง อาจใช้ภาพแบบเบลอเล็กน้อยใน Hero Section เพื่อให้ข้อความและ Search Box อ่านง่าย

## 4. Font

ใช้ฟอนต์ภาษาไทยที่อ่านง่าย เช่น:

* Prompt
* Noto Sans Thai
* Kanit

ควรใช้ขนาดตัวอักษรที่อ่านง่ายทั้งบน Desktop และ Mobile
หัวข้อควรชัดเจน ตัวหนาพอเหมาะ และมีระยะห่างที่ดี

## 6. UX/UI Requirements

* เว็บไซต์ต้องรองรับ Responsive Design ทั้ง Desktop, Tablet และ Mobile
* Search Box ต้องเด่น ใช้งานง่าย และเป็นจุดสำคัญของหน้าแรก
* Card ประกาศต้องอ่านง่าย มีรูปภาพชัดเจน และข้อมูลครบ
* ใช้ Rounded Corners และ Shadow แบบเบา ๆ
* มี Hover Effect บนปุ่มและ Card
* ปุ่มหลักใช้สี Teal
* ปุ่มรองใช้สีขาวขอบ Teal
* ใช้พื้นที่ว่างให้เหมาะสม ไม่อัดแน่นเกินไป
* Navigation ต้องใช้งานง่ายและสม่ำเสมอทุกหน้า
* Mobile Menu ต้องเป็น Hamburger Menu
* Footer ต้องเหมือนกันทุกหน้า

## 7. Code Requirements

เขียนโค้ดให้สะอาด เป็นระเบียบ และสามารถนำไปพัฒนาต่อได้ง่าย

ข้อกำหนดด้านโค้ด:

* แยกโครงสร้างไฟล์และหน้าเพจให้ชัดเจน
* ใช้ Component ที่สามารถ Reuse ได้
* ตั้งชื่อไฟล์ ตัวแปร และ Component ให้เข้าใจง่าย
* หลีกเลี่ยงการเขียนโค้ดซ้ำ
* แยกข้อมูล Mock Data ออกจาก UI Component
* เขียน CSS / Styling ให้เป็นระบบ
* ใช้ Design Tokens หรือ CSS Variables สำหรับสี ฟอนต์ ระยะห่าง และขนาด
* ทำให้รองรับการเชื่อมต่อ API ในอนาคต
* ทำให้รองรับการเพิ่มระบบ Login, Register, Favorite, Compare และระบบหลังบ้านในอนาคต
* โค้ดต้องอ่านง่าย มี Comment เฉพาะจุดที่จำเป็น
* หลีกเลี่ยง Inline Style ถ้าไม่จำเป็น
* ใช้ Semantic HTML เพื่อ SEO ที่ดี
* ตั้งค่า Meta Title และ Meta Description ให้เหมาะสม

## 15. Package Requirement Before Posting

ผู้ใช้ที่ Login แล้วจะยังไม่สามารถลงประกาศได้ทันที
ต้องสมัครหรือเลือกแพ็กเกจก่อน จึงจะสามารถลงขายหรือปล่อยเช่าคอนโดได้

### หน้า Packages Page

สร้างหน้า `/packages` สำหรับเลือกแพ็กเกจลงประกาศ

ต้องมีแพ็กเกจตัวอย่างอย่างน้อย 3 แบบ:

#### Free Package

* ราคา 0 บาท
* ลงประกาศได้ 1 รายการ
* ระยะเวลา 7 วัน
* รูปภาพสูงสุด 5 รูป
* ไม่มี Badge แนะนำ

#### Standard Package

* ราคา 299 บาท
* ลงประกาศได้ 5 รายการ
* ระยะเวลา 30 วัน
* รูปภาพสูงสุด 15 รูป
* แสดง Badge “แนะนำ”

#### Premium Package

* ราคา 599 บาท
* ลงประกาศได้ 15 รายการ
* ระยะเวลา 60 วัน
* รูปภาพสูงสุด 30 รูป
* ดันประกาศขึ้นด้านบน
* แสดง Badge “ประกาศพรีเมียม”

### เงื่อนไขการลงประกาศ

ก่อนเข้า `/post-property` ให้ตรวจสอบเงื่อนไข:

1. ถ้ายังไม่ได้ Login

   * Redirect ไป `/login`
   * แจ้งว่า “กรุณาเข้าสู่ระบบก่อนลงประกาศ”

2. ถ้า Login แล้ว แต่ยังไม่มี Package

   * Redirect ไป `/packages`
   * แจ้งว่า “กรุณาเลือกแพ็กเกจก่อนลงประกาศ”

3. ถ้า Login แล้ว และมี Package แล้ว

   * อนุญาตให้เข้า `/post-property`

4. ถ้า Package หมดอายุ หรือใช้จำนวนประกาศครบแล้ว

   * ไม่อนุญาตให้ลงประกาศเพิ่ม
   * แจ้งว่า “แพ็กเกจของคุณหมดอายุหรือใช้จำนวนประกาศครบแล้ว กรุณาต่ออายุแพ็กเกจ”

## 20. Auth Guard Requirement

ต้องมี Auth Guard เพื่อป้องกันหน้าที่ต้อง Login

### Auth Guard Logic

* ถ้า User Login แล้ว ให้เข้าได้
* ถ้ายังไม่ได้ Login ให้ Redirect ไป `/login`
* แสดงข้อความแจ้งเตือนว่า “กรุณาเข้าสู่ระบบก่อนใช้งานหน้านี้”

ตัวอย่าง Logic:

```ts
if (authService.isLoggedIn()) {
  return true;
} else {
  router.navigate(['/login']);
  return false;
}
```

## 21. Package Guard Requirement

ต้องมี Package Guard เพื่อป้องกันการลงประกาศโดยไม่มีแพ็กเกจ

### Package Guard Logic

* ถ้า User มี Active Package ให้เข้า `/post-property` ได้
* ถ้ายังไม่มี Package ให้ Redirect ไป `/packages`
* แสดงข้อความว่า “กรุณาเลือกแพ็กเกจก่อนลงประกาศ”

ตัวอย่าง Logic:

```ts
if (packageService.hasActivePackage()) {
  return true;
} else {
  router.navigate(['/packages']);
  return false;
}
```
